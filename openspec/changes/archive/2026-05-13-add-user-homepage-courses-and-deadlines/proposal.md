## Why

Сейчас у пользователя нет единой главной страницы, где можно быстро увидеть все свои курсы и ближайшие дедлайны. Курсовые карточки и календарь дедлайнов уже существуют по отдельности, поэтому имеет смысл собрать из них полезный dashboard как основную точку входа после авторизации.

## What Changes

- Обновить главную страницу авторизованного пользователя так, чтобы на ней одновременно отображались блок с курсами пользователя и блок с календарем дедлайнов.
- Переиспользовать существующие компоненты `CourseCard` и `DeadlinesCalendar` как основные строительные блоки страницы.
- Добавить структуру страницы и адаптивный layout, чтобы список курсов и календарь удобно читались на широких и узких экранах.
- Определить поведение страницы при отсутствии курсов или дедлайнов, чтобы главная страница оставалась информативной.

## Capabilities

### New Capabilities
- `user-homepage`: Главная страница авторизованного пользователя показывает все курсы пользователя и календарь дедлайнов в одном интерфейсе.

### Modified Capabilities
- None.

## Impact

- Affected code: `src/routes/_authenticated/index.tsx`, `src/course/*`, `src/deadlines-calendar/*`.
- UI impact: меняется основная страница после входа пользователя.
- API/dependencies: новые внешние зависимости не требуются; допускается использование текущих mock-источников данных до появления реального API.
